# 📚 书签同步工具 - 快速上手指南

## 🚀 5分钟快速开始

### 1. 启动应用
```bash
cd bookmark-sync-tool
npm start
```

### 2. 配置浏览器路径
1. 点击"自动检测"按钮
2. 工具会自动找到Chrome和Atlas的书签文件
3. 如果检测失败，可以手动"浏览"选择文件

### 3. 开始同步
1. 选择同步方向（推荐：双向同步）
2. 点击"🚀 启动自动同步"
3. 工具开始监控书签文件变化

### 4. 测试同步
1. 在Chrome中添加一个新书签
2. 观察同步日志显示检测和同步过程
3. 点击"🔄 重启 Atlas"按钮
4. 在Atlas中查看新书签

## 📋 主要功能

### 自动同步
- **实时监控**: 自动检测书签文件变化
- **智能同步**: 只同步有差异的书签
- **详细日志**: 显示同步过程和结果

### 手动操作
- **手动同步**: 随时执行一次性同步
- **一键重启**: 自动重启Atlas浏览器
- **状态检查**: 查看书签统计和差异

### 配置选项
- **同步方向**: 
  - 双向同步 (推荐)
  - Chrome → Atlas
  - Atlas → Chrome

## 🔧 常见问题

### Q: Atlas路径检测失败怎么办？
**A**: 
1. 确认Atlas已正确安装
2. 点击"帮助"查看可能的路径位置
3. 手动浏览到正确的Bookmarks文件

### Q: 同步后看不到新书签？
**A**: 
1. 点击"🔄 重启 Atlas"按钮
2. Atlas重启后应该能看到新书签
3. 如果仍有问题，检查同步日志

### Q: 如何验证同步是否成功？
**A**: 
1. 查看同步日志中的详细信息
2. 使用"手动同步一次"查看差异分析
3. 重启浏览器后检查书签

## 💡 使用技巧

### 最佳实践
1. **保持应用运行**: 让同步工具在后台运行以实现实时同步
2. **定期手动同步**: 偶尔执行手动同步验证一致性
3. **观察日志**: 通过日志了解同步状态和问题

### 高效工作流
```
1. 启动工具 → 2. 自动检测路径 → 3. 启动自动同步 → 4. 正常使用浏览器
```

### 故障排除
1. **重启应用**: 遇到问题时首先尝试重启同步工具
2. **检查路径**: 确认浏览器路径配置正确
3. **查看日志**: 日志中通常包含问题的详细信息

## 🎯 界面说明

### 主要按钮
- **🚀 启动自动同步**: 开始实时监控和同步
- **⏹️ 停止同步**: 停止自动同步
- **🔄 手动同步一次**: 执行一次性同步
- **🔄 重启 Atlas**: 自动重启Atlas浏览器

### 状态指示
- **运行中**: 绿色，自动同步正在工作
- **已停止**: 灰色，自动同步已停止
- **出错**: 红色，同步过程中出现错误

### 日志区域
- **蓝色**: 信息性消息
- **绿色**: 成功操作
- **红色**: 错误信息
- **橙色**: 警告信息
